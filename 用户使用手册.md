## 1. 系统概述

本系统用于采集海外三家博物馆的中国文物数据，经过清洗、翻译后存入 MySQL 数据库，并构建 Neo4j 知识图谱进行可视化展示。

### 数据来源

| 博物馆 | 英文名 | 数据量 |
|--------|--------|--------|
| 旧金山亚洲艺术博物馆 | Asian Art Museum of San Francisco | 748 件 |
| 大都会艺术博物馆 | The Metropolitan Museum of Art | 2,995 件 |
| 费城艺术博物馆 | Philadelphia Museum of Art | 2,500 件 |

### 处理流水线

```
爬取(spider) → 增量爬取(incremental) → 清洗(clean) → 翻译(translate) → MySQL入库(database) → Neo4j知识图谱(neo4j)
```

## 2. 环境准备

### 2.1 Python 环境

建议使用 Python 3.8+，安装以下依赖：

```bash
pip install requests beautifulsoup4 playwright pandas numpy pymysql tqdm
```

安装 Playwright 浏览器（仅旧金山亚洲艺术博物馆爬虫需要）：

```bash
playwright install chromium
```

### 2.2 数据库环境

- **MySQL 5.7+**：需提前创建数据库，字符集为 `utf8mb4`
- **Neo4j 4.x+**：需启动服务并开放 Bolt 端口（默认 7687）

### 2.3 API 密钥

翻译功能需要百度翻译 API，请提前申请并获取：
- `APP_ID`
- `SECRET_KEY`

申请地址：https://fanyi-api.baidu.com/

### 2.4 数据库连接配置

所有数据库上传脚本中均需填写以下配置（位于脚本顶部）：

```python
DB_HOST = ""    # MySQL 主机地址
DB_PORT = 0     # MySQL 端口
DB_USER = ""    # MySQL 用户名
DB_PASS = ""    # MySQL 密码
DB_NAME = ""    # 数据库名（默认 seitem）
```

## 4. 增量爬取

系统新增了公共增量爬取框架 `spider/incremental.py`，三个博物馆各有增量版爬虫。首次运行时执行全量爬取，后续运行仅处理新增和更新的记录，避免重复爬取已有数据。

### 4.1 运行方式

```bash
# 亚洲艺术博物馆（需浏览器交互）
python "spider/Asian Art Museum of San Francisco/spider_incremental.py"

# 大都会博物馆
python spider/daduhui/spider_incremental.py

# 费城博物馆
python spider/philamuseum/spider_incremental.py
```

### 4.2 核心机制

**指纹比对**：为每条记录计算 MD5 内容哈希，与历史指纹比对，判断记录状态：

| 状态 | 说明 |
|------|------|
| new | 新增记录，指纹库中不存在 |
| updated | 记录已存在但内容发生变化（哈希值不同） |
| unchanged | 记录已存在且内容未变（哈希值相同） |

**变更日志**：每次爬取自动记录以下信息：
- 爬取时间
- 新增 / 更新 / 未变数量
- 变更 ID 列表

**增量写入**：仅追加新增和更新的记录；更新记录会替换原文件中对应行。

### 4.3 生成的文件

| 文件 | 说明 |
|------|------|
| `{博物馆目录}/.crawl_fingerprints.json` | 指纹文件，存储每条记录的 MD5 哈希 |
| `{博物馆目录}/crawl_log.csv` | 变更日志，记录每次爬取的统计信息 |

### 4.4 各博物馆唯一标识

增量爬取依赖唯一标识来判断记录是否已存在：

| 博物馆 | 唯一标识字段 |
|--------|-------------|
| 亚洲艺术博物馆 | `detail_url` |
| 大都会博物馆 | `Object URL` |
| 费城博物馆 | `UUID` |

## 5. 数据清洗

### 5.1 大都会博物馆数据清洗

**脚本位置**：`clean/daduhui/dataclean(1).py`

**运行方法**：

```bash
cd clean/daduhui
python "dataclean(1).py"
```

**输入文件**：`china_met_museum_data.csv`（需放在当前目录）

**输出文件**：
- `cleaned_data.csv`：全部清洗后数据
- `cleaned_data_with_images.csv`：仅包含有图像的记录

**处理步骤**：
1. 空值填充（Title/Period/Medium/Artist 填 `'unknown'`，Image 填 `'no_image'`）
2. 字符串标准化（去空格 + 转小写）
3. 变体统一（`'unknown '` / `'Unknown'` → `'unknown'`）
4. 去除完全重复行
5. 筛选有图记录
6. 添加连续 Object ID

**去重策略**：基于详情页 URL（`Object URL`）去重，同一 URL 仅保留首条记录。

**type 字段自动推断**：清洗脚本会根据 `material` 字段自动推断文物的 `type`（类型），推断规则如下：

| material 包含关键词 | 推断 type |
|---------------------|-----------|
| ceramic / porcelain / pottery / stoneware / earthenware | Ceramics |
| bronze / iron / gold / silver / copper / brass / steel | Metalwork |
| silk / textile / fabric / tapestry / embroidery | Textiles |
| jade / nephrite | Jade |
| lacquer | Lacquer |
| enamel / cloisonné | Decorative Arts |
| painting / scroll | Paintings |
| glass | Glass |

若 material 不匹配任何关键词，则 type 保持原值或设为 `'unknown'`。

## 8. Neo4j 知识图谱

### 8.1 数据准备

将 `neo4j/` 目录下的 CSV 文件复制到 Neo4j 服务器的 `import` 文件夹中：

- `Asian Art Museum of San Francisco/` 下的 3 个 CSV 文件
- `philamuseum & daduhui/` 下的 3 个 CSV 文件

### 8.2 导入数据

连接 Neo4j 数据库（浏览器访问 `http://<服务器IP>:7474/`），按 `neo4j/Import_neo4j.md` 中的顺序执行 Cypher 语句：

**步骤 1 — 创建唯一约束**：

```cypher
CREATE CONSTRAINT artifact_id_unique FOR (a:Artifact) REQUIRE a.object_id IS UNIQUE;
CREATE CONSTRAINT museum_id_unique FOR (m:Museum) REQUIRE m.id IS UNIQUE;
CREATE CONSTRAINT dynasty_id_unique FOR (d:Dynasty) REQUIRE d.id IS UNIQUE;
CREATE CONSTRAINT artist_id_unique FOR (art:Artist) REQUIRE art.id IS UNIQUE;
```

**步骤 2 — 导入文物节点**：

```cypher
:auto
CALL {
  LOAD CSV WITH HEADERS FROM 'file:///artifacts.csv' AS row
  MERGE (a:Artifact {object_id: row.object_id})
  ON CREATE SET
    a.title_zh = row.title_zh,
    a.title_en = row.title_en,
    a.material = row.material,
    a.type = row.type,
    a.image_url = row.image_url
} IN TRANSACTIONS OF 2000 ROWS;
```

**步骤 3 — 建立"收藏"关系（Artifact → Museum）**

**步骤 4 — 建立"年代"关系（Artifact → Dynasty）**

**步骤 5 — 建立"作者"关系（Artifact → Artist）**

详细 Cypher 语句见 `neo4j/Import_neo4j.md`。

### 8.3 验证与浏览

导入完成后，执行以下查询验证关系网络：

```cypher
MATCH (m:Museum)<-[r1:COLLECTED_BY]-(a:Artifact)-[r2:BELONGS_TO]->(d:Dynasty)
OPTIONAL MATCH (a)-[r3:CREATED_BY]->(art:Artist)
RETURN m, r1, a, r2, d, r3, art
LIMIT 200
```

> 由于服务器性能限制，单次查询建议不超过 1000 条，可通过修改 `SKIP` 和 `LIMIT` 参数分页浏览。

### 8.4 知识图谱结构

**节点类型**：

| 节点 | 属性 |
|------|------|
| Museum | id, name |
| Artifact | object_id, title_zh, title_en, material, type, image_url |
| Dynasty | id, name_zh |
| Artist | id, name_zh |

**关系类型**：

| 关系 | 方向 | 说明 |
|------|------|------|
| COLLECTED_BY | Artifact → Museum | 文物被博物馆收藏 |
| BELONGS_TO | Artifact → Dynasty | 文物属于某朝代 |
| CREATED_BY | Artifact → Artist | 文物由某艺术家创作 |

## 11. 常见问题

### Q1：旧金山亚洲艺术博物馆爬虫遇到验证码怎么办？

脚本会自动暂停并提示。在浏览器窗口中手动完成验证码，然后在终端按回车继续。

### Q2：爬虫中途中断，如何恢复？

- **旧金山亚洲艺术博物馆**：支持断点续爬，直接重新运行脚本即可
- **大都会博物馆**：重新运行脚本，但会从头开始（中间结果已保存在 `1.csv` 中）
- **费城博物馆**：支持去重续爬，已有数据不会被重复写入

### Q3：增量爬取和全量爬取有什么区别？

- **全量爬取**：每次运行都重新获取所有数据，适用于首次采集
- **增量爬取**：首次运行全量爬取，后续运行仅处理新增和更新的记录，通过指纹比对判断变更状态，效率更高

### Q4：百度翻译 API 余额不足怎么办？

亚洲艺术博物馆和费城博物馆的翻译脚本会自动检测余额不足（错误码 54004）并保存进度。充值后重新运行脚本，会从断点处继续翻译。

### Q5：数据库上传脚本报连接错误？

检查以下配置是否正确填写：
- `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASS`、`DB_NAME`
- MySQL 服务是否已启动
- 是否已执行建表 SQL（`database/Sql-create.md`）

### Q6：Neo4j 导入时提示节点重复？

确保已先执行唯一约束创建语句（步骤 1）。使用 `MERGE` 而非 `CREATE` 可避免重复导入。

### Q7：CSV 文件编码问题？

- 旧金山亚洲艺术博物馆的原始爬取文件为 GBK 编码
- 其他文件均为 UTF-8 或 UTF-8-BOM 编码
- 清洗脚本已处理编码转换，无需手动操作

### Q8：数据库字段超长导致写入失败？

费城博物馆的 `upload_all.py` 和 `upload_artifacts.py` 内置了 VARCHAR 截断保护，会自动截断超长字段。其他上传脚本可能需要手动检查字段长度。

### Q9：增量爬取的指纹文件可以删除吗？

可以删除。删除 `.crawl_fingerprints.json` 后，下次运行增量爬取将视为首次运行，执行全量爬取并重新生成指纹文件。如需强制全量重新爬取，删除指纹文件即可。

### Q10：清洗脚本的去重逻辑是什么？

三个博物馆的清洗脚本统一使用**详情页 URL** 进行去重，同一详情页 URL 仅保留首条记录。相比之前的 Title+Image 组合键去重，基于 URL 去重更准确，能有效避免同一文物因标题或图片微小差异而被重复保留。
