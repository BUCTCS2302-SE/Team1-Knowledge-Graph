# 个人总结 — 严励（多源数据采集）

---

## 一、工作概述

在本项目中，我负责多源数据采集模块的设计与实现，完成了三家海外博物馆（大都会艺术博物馆、亚洲艺术博物馆、费城艺术博物馆）的中国文物数据爬取工作，共采集原始数据 **25,454 条**，为后续数据清洗、翻译和知识图谱构建提供了完整的数据基础。此外，我还设计并实现了增量爬取框架，支持对新增和更新文物的自动识别与采集。

---

## 二、主要工作内容

### 2.1 三馆爬虫系统开发

针对三家博物馆不同的网站架构和 API 接口，分别设计了差异化的爬取策略：

#### （1）大都会艺术博物馆爬虫

- **技术方案**：基于 Met Museum 开放 API（`collectionapi.metmuseum.org`），通过关键词搜索获取中国文物 ObjectID 列表，再逐条请求详情接口
- **核心实现**：
  - 使用 `requests` 库发送 REST API 请求，支持超时重试（timeout=10s）
  - 双关键词搜索策略（"China" + "Chinese"），通过 `set()` 去重确保结果完整
  - 限速策略：每 5 次请求暂停 1 秒，避免触发反爬机制
  - 断点续传：每 10 条数据追加写入 CSV，异常中断后可从断点继续
  - 失败记录追踪：单独记录请求失败的 ObjectID，便于后续补爬
- **采集成果**：原始数据 **19,901 条**

#### （2）亚洲艺术博物馆爬虫

- **技术方案**：基于 Playwright 浏览器自动化 + BeautifulSoup 解析，应对动态渲染页面和验证码拦截
- **核心实现**：
  - 使用 Playwright 启动 Chromium 浏览器，模拟真实用户行为
  - 自定义 User-Agent 伪装，降低被识别为爬虫的风险
  - 验证码人机协作机制：检测到验证码拦截时暂停并提示人工处理，处理完成后自动继续
  - 分页遍历策略：从列表页提取详情页链接，逐一访问解析
  - 随机延迟（1.2~2 秒），模拟人类浏览节奏
  - CSV 断点续写：通过 `get_last_index()` 读取已爬取进度，支持中断恢复
- **采集成果**：原始数据 **3,053 条**

#### （3）费城艺术博物馆爬虫

- **技术方案**：基于 POST API + 多线程并发，高效获取文物列表和详情
- **核心实现**：
  - 使用 `requests.Session` + `HTTPAdapter` 配置自动重试机制（最多 3 次，指数退避）
  - `ThreadPoolExecutor`（10 线程）并发获取文物详情，大幅提升采集效率
  - UUID 去重机制：通过 `processed_uuids` 集合避免重复处理
  - 失败重试：单独维护 `failed_uuids` 集合，每页爬取后自动重试失败项
  - IIIF 图片链接处理：自动拼接相对路径为完整 URL
- **采集成果**：原始数据 **2,500 条**

### 2.2 增量爬取框架设计与实现

为解决全量爬取效率低、重复下载的问题，设计并实现了通用增量爬取框架（`spider/incremental.py`），包含三个核心类：

| 类名 | 功能 | 技术实现 |
|------|------|---------|
| IncrementalTracker | 指纹比对 | 基于记录内容生成 MD5 哈希，与历史指纹比对，判断新增/更新/未变 |
| CrawlLogger | 变更日志 | 每次爬取追加记录时间、新增/更新/未变数量、变更 ID 列表 |
| IncrementalCSVWriter | 增量写入 | 仅追加新增记录，更新记录替换原文件对应行，未变记录跳过 |

**设计亮点**：
- 指纹文件（`.crawl_fingerprints.json`）持久化存储，支持跨会话比对
- `is_seen()` 方法支持快速判断记录是否已存在，无需计算哈希
- 变更日志（`crawl_log.csv`）支持审计追溯，可查看任意历史爬取详情
- 三馆各有适配的增量版爬虫（`spider_incremental.py`），首次运行全量爬取，后续仅处理变更

## 三、技术难点与解决方案

| 难点 | 解决方案 |
|------|---------|
| 亚洲艺术博物馆验证码拦截 | 设计人机协作机制，检测到拦截时暂停等待人工处理，处理后自动继续 |
| 大都会 API 限速 | 每 5 次请求暂停 1 秒，失败自动跳过并记录 |
| 费城博物馆高并发稳定性 | Session 级重试 + 失败 UUID 集合 + 自动重试机制 |
| 爬取中断恢复 | 所有爬虫均支持断点续传，CSV 追加写入 |
| 全量爬取效率低 | 设计增量爬取框架，基于 MD5 指纹比对，仅处理变更记录 |
