## 一、工作概述

在本项目中，我负责数据预处理与质量保障模块的设计与实现，涵盖三家博物馆（大都会、亚洲艺术、费城）的数据清洗、翻译和质量检测工作。通过多步骤清洗流水线，将原始爬虫数据 **25,454 条** 精炼为高质量数据 **6,243 条**；通过百度翻译 API 实现中英双语对齐；通过 106 项自动化测试验证数据质量，核心功能通过率达 **100%**。

## 二、主要工作内容

### 2.1 数据清洗流水线

为三家博物馆分别设计了定制化的清洗脚本，统一遵循以下流水线架构：

```
原始数据 → 完全重复去重 → 无效记录过滤 → 关键字段完整性检查 → 详情页URL去重 → 朝代匹配 → 标准化输出
```

#### （1）大都会博物馆清洗（`clean_daduhui.py`）

- **清洗步骤**：完全去重 → 去除无图片 → 关键字段完整性 → 详情页URL去重 → 朝代匹配 → type推断
- **核心创新 — type 字段智能推断**：大都会 API 未提供 type 字段，原始数据 type 列 100% 为空。设计了基于 material（材质）关键词的智能推断引擎，覆盖 9 大文物类型（Ceramics、Metalwork、Textiles、Jade、Lacquer、Paintings、Glass、Decorative Arts 等），推断覆盖率达 **99.8%**
- **朝代匹配**：支持多种 Period 格式解析（"1800–1900"、"ca. 1897"、"18th century"、"late 18th–early 19th century" 等），通过年份中值映射到中国朝代 ID
- **清洗成果**：19,901 → 2,995 条（保留率 15.0%）

#### （2）亚洲艺术博物馆清洗（`clean_asian.py`）

- **清洗步骤**：完全去重 → 仅保留 Chinese Art 部门 → 去除无图片 → 详情页URL去重 → 字段完整性+Date有效性 → 朝代匹配
- **核心创新 — 非中国时期过滤**：设计了 `NON_CHINESE_PERIODS` 关键词表（含 Edo、Muromachi、Joseon 等日韩时期），自动过滤非中国文物
- **Date 清洗**：去除 "approx."、"ca."、"before" 等前缀，统一日期格式
- **朝代匹配**：支持 Dynasty/Period 字段直接匹配 + Date 年份解析双重策略
- **清洗成果**：3,053 → 748 条（保留率 24.5%）

#### （3）费城艺术博物馆清洗（`clean_philamuseum.py`）

- **清洗步骤**：完全去重 → 去除无图片/详情链接 → 关键字段完整性 → 详情页URL去重 → HTML标签清洗 → 朝代匹配
- **核心创新 — HTML 标签清洗**：费城数据摘要字段包含大量 HTML 标签（`<p>`、`<br>`），设计了 `clean_html()` 函数，将 HTML 转为纯文本
- **朝代匹配**：基于关键词优先长匹配策略（如 "Northern Song" 优先于 "Song"），避免误匹配
- **清洗成果**：2,500 → 2,500 条（保留率 100%，原始数据质量较高）

### 2.2 统一去重策略

将三家博物馆的清洗脚本统一改为基于详情页 URL 去重，确保去重逻辑一致：

| 清洗脚本 | 去重字段 |
|----------|----------|
| `clean_asian.py` | `detail_url` |
| `clean_daduhui.py` | `Object URL` |
| `clean_philamuseum.py` | `详情链接` |

### 2.3 数据翻译

基于百度翻译 API，为三家博物馆的数据实现英中翻译：

- **翻译字段**：title、type、material、description、artist 等
- **批量翻译**：支持多句拼接换行文本一次性翻译，提升 API 调用效率
- **断点续传**：加载已有翻译进度文件，中断后可从断点继续
- **翻译成果**：亚洲艺术和费城核心字段翻译率均达 **100%**

### 2.4 数据对齐保障

将 `clean/daduhui/daduhui_final.csv` 与 `translate/daduhui/daduhui_final.csv` 基于 `detail_url` 严格对齐，确保两个文件行数一致、顺序一致、object_id 从 2501 连续编号。

## 三、技术难点与解决方案

| 难点 | 解决方案 |
|------|---------|
| 大都会 type 字段 100% 为空 | 设计基于 material 的智能推断引擎，推断覆盖率 99.8% |
| Period 格式多样（世纪、年份范围、缩写） | 多格式正则解析器，支持 ca./late/early 等修饰词 |
| 亚洲艺术含非中国文物 | 设计 NON_CHINESE_PERIODS 过滤器，自动排除日韩时期 |
| 费城数据含 HTML 标签 | clean_html() 函数，将 HTML 转为纯文本 |
| 翻译中断需重新开始 | 实现断点续传，加载已有翻译进度 |
| 清洗-翻译数据行数不一致 | 基于 detail_url 严格对齐，确保数据一致性 |
| 质量检测手工效率低 | 106 项自动化测试框架，一键检测所有维度 |
